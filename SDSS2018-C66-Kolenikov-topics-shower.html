Task at hand

One of the authors is serving as a Program Chair of the Survey Methods Research Section of the ASA.

Computers must process the data, while humans should spend time thinking… wouldn’t it be great if computers could analyze the text of the abstracts and create the conference program??

Scope

  • Scrape 10 years of JSM abstracts (2008 – 2017)
  • Pre-process text data for analysis
  • Descriptive text modeling (NMF, LDA)
  • Expert-informed classification of topics

Data preprocessing

  1. Data Cleaning
    • remove all observations with no content in abstract text
    • include only observations sponsored by SRMS
  2. Tokenize text into individual words (aka tokens)
  3. Lemmatize text
    • analyzing, analyze, analyzes -> analyz*
  4. Remove stopwords
    • the, of, are, is, thereby, however, etc
    • additional stopwords for this project: data, roundtable

Vectorize text

Convert text data into its numeric representation

  1. Frequency Count -> raw counts of words
    • Used in Latent Dirichlet Allocation
  2. Term Frequency -> Inverse Document Frequency (TF-IDF)
    • Used in Non-negative Matrix Factorization

Histograms

Non-negative Matrix Factorization (NMF)

  • Unsupervised learning algorithm intended to extract common features
  • Represented as a weighted sum of some components
  • Product of document-term matrix and topic-term matrix to approximate document-topic matrix
  • Another name for spectral embedding (??)

Visualization of Results

  • Used visualization algorithm, t-SNE, to transform data into 2-D representation
    • t-distributed stochastic neighbor embedding is a manifold learning algorithm used in visualization and exploratory analysis
  • The colors represent the most dominant topic for a given abstract
    • remember that each abstract can have several topics associated with it but not all will be captured in a 2-D scatterplot

Latent Dirichlet Allocation (LDA)

  • Groups of words that frequently appear together indicate a topic
  • Each document contains a mixture of topics
  • Probabilistic model using P(word|topic) to obtain P(topic|document)
    • underlying EM-algorithm: probabilities are calculated iteratively until convergence

LDA viz (hint: hover over/click on circles to see which terms are associated with the topics)

Limitations

  • Unsupervised machine learning results should be taken with a grain of salt
  • “Topics” as uncovered by machine learning models may not be the same as how humans understand a coherent topic
    • For machine learning models, “topics” are comprised of components which may or may not have semantic meaning
    • Different author writing styles may be picked up as different topics
  • Pre-processing data and model parameters may dramatically change the separation of topics

Abstract similarity

  • the more symbols the two codes share, the closer the topics are
  • the more codes the two abstracts share, the closer they are
  • similarity =
    • 1, if two codes are identical
    • a1, if two codes share the stem up to the dot
    • a2, if two codes share the two letters
    • a3, if two codes share the first letter only
    • 0, if two codes have different first letter

Tentative workflow

  • Assign codes to keywords (~500)
  • Assign codes to existing abstracts
  • Tune parameters of the similarity metric based on historic clustering of abstracts to sessions
  • Apply the parameters to the current (2018) abstracts to gauge their similarity
  • Cluster 6-7 abstracts per contributed session

Tuning the parameters

  • maximize [ave within session similarity]/[ave between session similarity]
  • quasi-Monte Carlo regular sequence of parameter values
  • highest within/between session similarity ratio:
    • a1 = 0.552
    • a2 = 0.074
    • a3 = -0.226

Clustering new abstracts

  • Convert keywords to codes
  • Extract keywords from title, convert to codes
    • Abstract text can be used, too
  • Compute similarity measures using the optimal parameters
  • Run a clustering algorithm
    • Convert similarity to dissimilarity/distance
